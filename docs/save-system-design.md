# 存档系统设计文档

## 1. 概述

为 Web Minecraft Demo 实现存档/读档功能，让玩家可以保存游戏进度并在下次访问时继续游戏。

## 2. 需要保存的数据

| 数据 | 类型 | 说明 |
|------|------|------|
| 世界种子 | `number` | 用于地形生成 |
| 方块变更 | `BlockChange[]` | 玩家放置/破坏的方块 |
| 玩家位置 | `{x, y, z}` | 三维坐标 |
| 玩家视角 | `{yaw, pitch}` | 朝向 |
| 飞行状态 | `boolean` | 是否开启飞行 |
| 快捷栏选择 | `number` | 当前选中的方块索引 |

## 3. 设计决策（已确认）

| 决策项 | 方案 | 说明 |
|--------|------|------|
| 存储方案 | **IndexedDB** | 大容量，浏览器原生支持 |
| 存档策略 | **种子 + 增量差异** | 只保存变更方块，体积小 |
| 数据格式 | **纯 JSON** | 先保持简单，后续考虑压缩 |
| 存档槽位 | **3 个槽位** | 兼顾简洁和灵活性 |
| 自动存档 | **事件触发** | 方块变更后自动保存 |

## 4. 存档数据结构

```typescript
interface SaveData {
  version: 1;
  name: string;                  // "存档 1" / "存档 2" / "存档 3"
  timestamp: number;
  
  world: {
    seed: number;
    radius: number;
    changes: [number, number, number, BlockType][];  // [x, y, z, type]
  };
  
  player: {
    position: { x: number; y: number; z: number };
    yaw: number;
    pitch: number;
    flying: boolean;
    hotbarIndex: number;
  };
}
```

## 6. 风险与注意事项

- **IndexedDB 隐私模式**：部分浏览器隐私模式下可能受限
- **异步操作**：所有 IndexedDB 操作需 async/await
- **数据一致性**：保存时需确保游戏状态完整
- **存储限制**：单个域名通常 50MB+，足够使用
